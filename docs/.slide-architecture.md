# Docker 教程 Slide 架构规范

## 核心原则

**按内容主题组织，而非单个 Slide 文件**

一个教学主题可能需要多个 Slide 来完整展示，因此采用**主题文件夹**结构，每个文件夹包含该主题所需的所有子 Slide。

## 目录结构规范

```
slides/
├── 1_Title/                    # 编号_主题名
│   └── index.tsx               # 主题入口（简单主题可单文件）
│
├── 2_WhatIsDocker/             # 复杂主题：多文件
│   ├── index.tsx               # 主题入口，编排子 Slide 的 Sequence
│   ├── Definition.tsx          # 子 Slide：知识点1
│   ├── CoreConcepts.tsx        # 子 Slide：知识点2
│   └── Compare.tsx             # 子 Slide：知识点3
│
├── 3_BasicCommands/
│   ├── index.tsx
│   └── ...
│
└── index.ts                    # 统一导出所有主题
```

## 命名规范

| 类型 | 命名规则 | 示例 |
|------|---------|------|
| 主题文件夹 | `{序号}_{PascalCase主题名}` | `2_WhatIsDocker/` |
| 子 Slide 文件 | `{PascalCase知识点名}.tsx` | `CoreConcepts.tsx` |
| 组件导出名 | `{PascalCase}` | `CoreConcepts` |
| Scene 入口 | `{主题名}Scene` | `WhatIsDockerScene` |

## 两种切换模式

### 模式 A：Sequence 切换（Slide 之间）

适用场景：不同知识点之间需要**清晰隔离**，各自独立时间轴

```tsx
// index.tsx - 主题入口
export const WhatIsDockerScene: React.FC = () => {
  return (
    <AbsoluteFill>
      <Sequence from={0} durationInFrames={90}>
        <Definition />
      </Sequence>
      <Sequence from={90} durationInFrames={200}>
        <CoreConcepts />
      </Sequence>
      <Sequence from={290} durationInFrames={250}>
        <Compare />
      </Sequence>
    </AbsoluteFill>
  );
};
```

### 模式 B：帧内动画切换（Slide 内部）

适用场景：内容**连续性强**，需要流畅过渡

```tsx
// CoreConcepts.tsx - 单文件内帧数控制
export const CoreConcepts: React.FC = () => {
  const frame = useCurrentFrame();
  
  // 阶段1：镜像概念（0-60帧）
  // 阶段2：容器概念（60-120帧）
  // 阶段3：仓库概念（120-180帧）
  
  const showImage = frame < 60;
  const showContainer = frame >= 60 && frame < 120;
  const showRegistry = frame >= 120;
  
  return (
    <AbsoluteFill>
      {showImage && <ImageConcept />}
      {showContainer && <ContainerConcept />}
      {showRegistry && <RegistryConcept />}
    </AbsoluteFill>
  );
};
```

## 导出规范

### 主题入口文件 (index.tsx) 必须导出

```tsx
// 必须导出
export const WhatIsDockerScene: React.FC;           // 主组件
export const WHAT_IS_DOCKER_TOTAL_FRAMES: number;   // 总帧数

// 可选导出
export const SLIDE_DURATIONS: object;               // 各子 Slide 时长
export { Definition } from './Definition';          // 子组件（供复用）
```

### slides/index.ts 统一导出

```tsx
// 导出 Scene 组件
export { WhatIsDockerScene } from './2_WhatIsDocker';

// 导出总帧数（供主入口计算时长）
export { WHAT_IS_DOCKER_TOTAL_FRAMES } from './2_WhatIsDocker';

// 兼容旧命名（可选）
export { WhatIsDockerScene as WhatIsDockerSlide } from './2_WhatIsDocker';
```

## 主入口引用方式

```tsx
// DockerTutorial/index.tsx
import {
  WhatIsDockerSlide,
  WHAT_IS_DOCKER_TOTAL_FRAMES,
} from './slides';

const SLIDES_CONFIG = [
  { 
    name: 'WhatIsDockerSlide', 
    duration: WHAT_IS_DOCKER_TOTAL_FRAMES,  // 使用导出的总帧数
    component: WhatIsDockerSlide 
  },
  // ...
];
```

## 层级职责

| 层级 | 职责 | 切换方式 |
|------|------|---------|
| **主题文件夹** | 完整教学单元 | Sequence 切换 |
| **子 Slide** | 单个知识点 | 帧内动画（可选） |

## 决策指南

| 问题 | 答案 | 建议 |
|------|------|------|
| 内容是否需要分多个知识点讲解？ | 是 | 创建主题文件夹 |
| 知识点之间是否需要清晰隔离？ | 是 | 用 Sequence 切换 |
| 知识点内部是否需要分阶段展示？ | 是 | 用帧内动画切换 |
| 主题是否简单到单文件足够？ | 是 | 直接创建 index.tsx |

## 示例：简单主题 vs 复杂主题

### 简单主题（单文件）

```
1_Title/
└── index.tsx    # 所有内容在一个文件
```

### 复杂主题（多文件）

```
2_WhatIsDocker/
├── index.tsx           # 编排入口
├── Definition.tsx      # 子 Slide 1
├── CoreConcepts.tsx    # 子 Slide 2
└── Compare.tsx         # 子 Slide 3
```
